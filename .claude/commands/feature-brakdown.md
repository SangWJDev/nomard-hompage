  ## 분석 프레임워크

  새로운 기능 개발에 최적화된 구조화된 접근 방식으로 이 기능을 분석해주세요:

  ### 1. 이해 단계
  - **기능 개요**: 무엇을 만들어야 하는지 요약
  - **사용자 영향**: 누가 사용하며 어떤 문제를 해결하는가?
  - **범위 경계**: 명확하게 범위 안에 포함되는 것과 제외되는 것
  - **코드베이스 컨텍스트**: 참조할 관련 파일 및 컴포넌트 식별

  ### 2. 작업 분해
  기능을 구체적이고 테스트 가능한 서브태스크로 분해합니다. 각 서브태스크마다:
  - 명확하고 실행 가능한 설명
  - 구체적인 결과물
  - 완료 기준 정의

  ### 3. 우선순위 분석
  다음 프레임워크를 사용하여 우선순위 레벨을 할당합니다:

  - **🔴 Critical (필수)**: 기능이 작동하는 데 필요한 핵심 기능; 다른 작업을 블로킹함
  - **🟡 High (높음)**: 중요한 기능; 다른 작업과 병렬로 수행 가능
  - **🟢 Medium (중간)**: 보조 기능; 사용자 경험을 향상시킴
  - **⚪ Low (낮음)**: 있으면 좋은 기능; 선택적 개선사항

  ### 4. 의존성 매핑
  각 서브태스크에 대해 다음을 식별합니다:
  - **의존**: 먼저 완료되어야 하는 작업
  - **블로킹**: 이 작업을 기다리는 작업
  - **병렬**: 동시에 수행 가능한 작업

  의존성을 명확한 형식으로 표시:
  작업 A (Critical)
    └─> 작업 B (High) - A에 의존
    └─> 작업 C (High) - A에 의존
         └─> 작업 D (Medium) - C에 의존
  작업 E (High) - 병렬 실행 가능

  ### 5. 시간 추정
  각 서브태스크의 복잡도를 추정합니다:
  - **S** (Small): < 1시간 - 간단한 변경, 최소한의 테스트
  - **M** (Medium): 1-4시간 - 중간 복잡도, 테스트 필요
  - **L** (Large): > 4시간 - 복잡함, 추가 분해 필요할 수 있음

  전체 예상 시간 범위를 제공합니다.

  ### 6. 실행 계획
  최적의 구현 순서를 추천합니다:

  **1단계: 기반** (Critical 경로 작업)
  1. 작업 X [Critical, S, 의존성 없음]
  2. 작업 Y [Critical, M, X에 의존]

  **2단계: 핵심 기능** (높은 우선순위, 병렬화 가능)
  3. 작업 A [High, M, Y에 의존]
  4. 작업 B [High, L, Y에 의존]

  **3단계: 개선사항** (중간/낮은 우선순위)
  5. 작업 C [Medium, S, A, B에 의존]

  ### 7. 리스크 평가
  잠재적 과제를 식별합니다:
  - 기술적 리스크 (복잡도, 미지의 요소)
  - 통합 리스크 (변경사항 충돌, 호환성)
  - 제안된 완화 전략
